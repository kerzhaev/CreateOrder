# Спецификация: Улучшение UI поиска сотрудников

**Версия:** 1.0.0
**Дата:** 15.02.2026
**Автор:** Кержаев Евгений

## 1. Цель
Повысить удобство работы с формой `frmSearchFIO` за счет внедрения многоколоночного списка и навигации с клавиатуры.

## 2. Требования
### 2.1. Многоколоночный список (Multi-column ListBox)
* Компонент `lstResults` должен иметь 5 колонок:
  1. Личный номер (скрыт или узкий)
  2. ФИО (основной)
  3. Звание
  4. Должность
  5. Часть
* Данные должны заполняться через массив или свойство `.List()`, а не конкатенацией строк.

### 2.2. Живой поиск (Live Filtering)
* Поиск осуществляется по вхождению подстроки в ФИО или Личный номер.
* Регистронезависимый поиск.

### 2.3. Навигация (Keyboard Navigation)
* **В поле поиска (`txtSearch`):**
  * `Enter`: Если в списке 1 результат — выбрать его. Если больше — фокус переходит в список.
  * `Down Arrow`: Фокус переходит в список результатов.
* **В списке (`lstResults`):**
  * `Enter`: Выбор текущего сотрудника (аналог двойного клика).
