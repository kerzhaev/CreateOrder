# Спецификация: операторский интерфейс мастера зачисления

## Цель

Превратить технический мастер зачисления в понятную оператору форму: новая карточка открывается пустой, справочные значения выбираются из редактируемых таблиц, суммы подтягиваются автоматически, а проверки объясняют, что исправить.

## Подтверждённые решения

- При открытии «Мастера зачисления» создаётся чистая новая карточка. Существующая запись открывается только после явного выбора сотрудника/черновика.
- `Категория службы` — единственное поле вида службы: `CONTRACT`, `MOBILIZED`, `CONSCRIPT`. Поле `Признак контракта` удаляется из формы и больше не дублирует категорию.
- `Основание контракта` переименовывается в «Основание службы / контракта»; оно вводится только когда есть подтверждающий документ или условие, которое должно попасть в приказ.
- Воинское звание, ВУС, должность, подразделение, воинская часть, тарифный разряд, классность и секретность выбираются из справочников с возможностью печати для поиска.
- Оклады по званию и должности оператор не вводит: они только отображаются и рассчитываются из выбранных значений справочника.
- Справочник доступен оператору на отдельном листе `EnrollmentReferenceData`; изменение строк справочника не требует изменения VBA.
- Все даты проверяются как даты; ИНН, СНИЛС, серия/номер паспорта и код подразделения проверяются по формату до сохранения и экспорта.
- Ежемесячные выплаты визуально разделяются на «Приказ МО №727», «Приказ МО №430дсп» и «Индивидуальные выплаты». Проценты классности и секретности подставляются из выбранного уровня справочника.
- При создании новой карточки стандартные выплаты и их параметры отображаются сразу по настройкам по умолчанию; персональные данные и документы при этом остаются пустыми.
- Основания выплат поясняются: они выводятся в Word рядом с включённой выплатой. Раздел «Проверка и Word» показывает понятные блоки «что готово», «что исправить», «что попадёт в приказ».

## Не утверждённые нормативные размеры

Для классности, секретности, иных уровней ФИЗО и полного набора выплат приложение не предполагает проценты. В справочник добавляются редактируемые строки и демонстрационные неактивные заготовки; только подтверждённые значения используются автоматически.

## Приёмка

1. Новый запуск мастера не показывает данные предыдущей карточки.
2. Выбор звания и тарифного разряда автоматически показывает соответствующие оклады; ручное изменение окладов невозможно.
3. Оператор меняет справочник на листе, открывает новую форму и видит новое значение.
4. Некорректные дата, ИНН, СНИЛС или паспорт блокируют сохранение с понятным текстом ошибки.
5. Включённая выплата и её основание попадают в предпросмотр и Word; разные НПА показаны отдельными блоками.
