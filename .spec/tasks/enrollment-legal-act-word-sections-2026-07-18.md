# Нормативные разделы приказа о зачислении

## Цель

Word-приказ о зачислении должен повторять логику образца: выплаты не смешиваются
в общий список, а идут в отдельных результативных разделах по нормативному акту.

## Правила формирования

1. После кадрового пункта, окладов и льготной выслуги генератор создаёт раздел
   приказа МО РФ №727, только если выбрана хотя бы одна выплата этой группы.
   Раздел начинается полной преамбулой приказа и заканчивается списком выбранных
   выплат.
2. Затем, только при наличии выбранных выплат, создаётся самостоятельный раздел
   приказа МО РФ №430дсп с полной преамбулой и собственным списком. В этой группе:
   надбавка по должностям 1-4 тарифных разрядов, контрактная надбавка, классность,
   ФИЗО и особые достижения/медаль.
3. Секретность, надбавка по должности и особые условия остаются в группе №727.
   Льготная выслуга остаётся отдельным абзацем вне групп. Премия и разовые выплаты
   сохраняют самостоятельную форму из образца.
4. Для индивидуальных выплат (ФИЗО, классность, секретность, медаль, контрактная
   надбавка, премия и разовые выплаты) оператор вводит подтверждающий документ.
   Он выводится в тот же Word-абзац только при включённой выплате. Для автоматически
   назначаемой надбавки 1-4 тарифных разрядов подтверждающий документ не нужен.
5. Для каждой типовой выплаты на листе `Настройки` хранится настраиваемая ссылка
   на нормативный пункт. Она ставится перед текстом выплаты в Word, если заполнена.
   Значения по умолчанию задаются только для уже подтверждённых пунктов №430дсп;
   другие формулировки оператор может исправить в настройках.
6. Иная ежемесячная выплата использует поле `НПА / пункт` (ранее «Параметр»).
   Выплаты с одинаковым значением этого поля собираются под самостоятельной
   преамбулой «В соответствии с ... установить:». Так можно добавлять новые приказы
   без изменения VBA.

## Хранение

- В строке зачисления хранится только выбор выплаты, её размер/параметр/дата и,
  когда он нужен, текст подтверждающего документа.
- Полная преамбула приказа, принадлежность выплаты к группе и нормативный пункт
  являются общими настройками определения выплаты, а не повторяются в каждой
  карточке военнослужащего.
- Word — итоговое представление этих структурированных данных; Excel хранит
  достаточные сведения для повторного просмотра, проверки и повторного экспорта.

## Проверка

- Изолированный тест должен подтвердить: разделы №727 и №430дсп появляются только
  при выбранных выплатах; 1-4 тариф попадает в №430дсп и не требует основания;
  документ ФИЗО отображается при включении; иная выплата формирует свой НПА-раздел.
- После автоматической проверки модули импортируются в `CreateOrder.xlsm` с копией
  книги до импорта.
- До открытия рабочей книги установщик обязательно запускает тесты на временных
  копиях. Тестовый пробник использует `Option Explicit`, объявляет все переменные,
  и возвращает ошибки как текст, не переводя VBE в режим отладки.
