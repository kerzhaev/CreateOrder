# D89 Validation

## Goal
Добавить отдельную проверку внешнего шаблона `Д89` с подсветкой ошибок в самом файле и с накопительной историей прогонов на отдельном листе шаблона.

## Scope
- Новая кнопка Ribbon: `Проверить Д89`.
- Новый VBA-модуль `mdlZP12Validation`.
- Проверка идентификаторов, ФИО, звания, даты рождения, контракта, даты начала действия и пересечений периодов.
- Нормализация лишних пробелов в ФИО при сравнении.
- Если сотрудник однозначно разрешен хотя бы по одному идентификатору, строка проверяется на все доступные ошибки, а не только на первую найденную.
- История прогонов на листе `История_проверок_Д89`.
- Обновление документации и контекста проекта.
- PowerShell smoke/regression script `Test-ZP12Validation.ps1` для воспроизводимой двухпроходной проверки `NEW/OPEN/RESOLVED`.

## Стабильность расположения листов и тестов

- `mdlZP12Validation` находит штат только по имени листа `Штат`, а не по его порядковому номеру. Добавление служебных листов не должно менять результат проверки Д89.
- `Test-ZP12Validation.ps1` перед импортом VBA создаёт отдельную копию `CreateOrder.xlsm` в `_tmp_zp12_validation_test`; основная книга не используется как тестовый носитель и не сохраняется тестом.
- Проверка 15.07.2026: двухпроходный сценарий завершился `RESULT: PASS` (`NEW=7`, затем `OPEN=4`, `RESOLVED=3`); в рабочую книгу импортирован исправленный модуль, а исходный файл сохранён в `CreateOrderBackups\zp12-staff-sheet-name-20260715-202041\`.
