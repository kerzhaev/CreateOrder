# Спецификация: Гибридная система лицензирования (Time-Bomb + File-Based Activation)
## Описание
Система оффлайн-защиты с проверкой срока действия, HWID-привязкой для персональных ключей и файловым обменом между пользовательской книгой `CreateOrder.xlsm` и отдельным генератором лицензий.

## Требования
- До даты `PUBLIC_EXPIRY_DATE` (вшита в код) программа работает без обязательной активации.
- После истечения требуется персональный ключ (привязка к HWID) или корпоративный ключ без привязки к ПК.
- В `frmAbout` пользователь должен видеть понятный статус лицензии и причину отказа при неудачной активации.
- Из `CreateOrder.xlsm` пользователь должен уметь сохранить текстовый файл-запрос на активацию с техническими данными:
  - версия программы;
  - имя файла;
  - текущая дата/время системы;
  - HWID;
  - код текущего состояния лицензии;
  - маска уже сохраненного ключа;
  - рекомендации по активации.
- Из `CreateOrder.xlsm` пользователь должен уметь загрузить готовый текстовый файл-ответ с кодом активации, не копируя ключ вручную.
- Генератор лицензий должен уметь открыть файл-запрос, автоматически подставить HWID и предлагать выпуск ответного текстового файла.
- Генератор лицензий должен сохранять два формата:
  - обычный текстовый пакет с кодом;
  - файл-ответ активации, пригодный для прямой загрузки в `CreateOrder.xlsm`.
- Ошибки активации должны разделяться как минимум на:
  - неверный формат ключа;
  - просроченный ключ;
  - неподходящий HWID / персональный ключ не для этого ПК;
  - нераспознанный корпоративный ключ;
  - защитная блокировка по дате.
- Должна сохраняться защита от перевода системных часов.
