# Спецификация: единый мастер кадровых действий

## Цель

Оператор работает с кадровыми действиями через единый понятный вход, а не через технические листы и список макросов Excel.

Поддерживаются три сценария:

1. зачисление;
2. перемещение;
3. исключение из списков личного состава.

## Принципы

- Выбор сценария делается в мастере кадровых действий; тип события не вводится вручную в ячейку.
- Для зачисления используется существующий подробный мастер `frmEnrollmentWizard` и существующий пакетный Word-экспорт. Успешное сохранение зачисления обязательно создаёт внутреннюю карточку и событие `ENROLLMENT`; повторное сохранение той же карточки не создаёт дубль.
- Для перемещения и исключения используется кадровая форма, заранее подготовленная выбранным типом действия; технический лист не должен быть обычной точкой входа оператора.
- Все три сценария создают или используют одну внутреннюю карточку сотрудника, неизменяемый журнал `PersonnelEvents`, снимки состояний, назначения выплат и `DocumentRegistry`. Исключение из списков устанавливает `IsActive = NO` в карточке `Employees`, сохраняет снимок состояния после события и прекращает активные назначения выплат.
- После сохранения мастер показывает `EventID` и только доступные для выбранного события действия: сформировать приказ, открыть историю, начать следующее действие.
- Зачисление экспортируется через существующий поток `ExportSelectedEnrollmentPackage` и сразу связывает Word-файл с созданным `ENROLLMENT` в `DocumentRegistry`; перемещение и исключение — через `ExportPersonnelEventOrder`.
- Нельзя предлагать экспорт приказа перемещения/исключения до успешного сохранения события. Нельзя предлагать кадровый экспорт для зачисления через поток, который его не поддерживает.

## Состав первой поставки

- Команды ленты в группе «Кадровые действия»: «Зачисление», «Перемещение», «Исключение», «История сотрудника». Сохранение и экспорт перемещения/исключения выполняются только из открытого кадрового мастера.
- Публичные команды подготовки каждой формы без списка макросов.
- Для перемещения и исключения подготовка формы устанавливает нужный тип события и очищает остатки несвязанного предыдущего сценария.
- Для перемещения и исключения после сохранения доступна команда формирования Word-приказа по явно сохранённому `EventID`.
- Пользовательская документация описывает единый маршрут и отделяет его от диагностических листов.

## Вторая поставка: кадровый мастер

- Для `TRANSFER` и `EXCLUSION` создаётся форма `frmPersonnelActionWizard`; оператор больше не должен открывать `PersonnelEventInput` или запускать макрос из списка Excel.
- Форма принимает внутренний ID сотрудника, даты события и вступления в силу, реквизиты и основание приказа, новое состояние, служебные даты и сведения о месте убытия.
- Команда «Загрузить состояние» заполняет поля текущими подтверждёнными данными сотрудника. Сохранение записывает событие только после явного нажатия «Сохранить».
- После сохранения форма показывает `EventID` и позволяет сформировать Word-приказ или открыть историю. До сохранения эти действия недоступны.
- Ошибки ввода и результат сохраняются в строке состояния и на форме; штатные действия не открывают ненужные `MsgBox`.
- Зачисление остаётся отдельным подробным `frmEnrollmentWizard`: он уже содержит существенно более широкий состав полей и пакетный Word-экспорт. Лента объединяет все три сценария в одной группе.

## Приёмка

1. С ленты открывается нужный сценарий без `Alt+F8`.
2. Зачисление открывает мастер зачисления; перемещение и исключение — соответствующую подготовленную кадровую форму без перехода на технический лист.
3. После сохранения перемещения или исключения команда экспорта создаёт Word-проект и запись в `DocumentRegistry`.
4. Выбор одного сценария не может случайно сохранить тип или данные предыдущего независимого сценария.
5. Изолированная приёмка проходит сквозную цепочку `ENROLLMENT → TRANSFER → EXCLUSION` через пользовательский мастер для двух последних событий, создаёт Word-проекты обоих кадровых действий и подтверждает, что после исключения карточка становится неактивной.
6. Ручное сохранение и экспорт из мастера зачисления создаёт ровно одну карточку, ровно одно событие `ENROLLMENT` и связанную запись `ENROLLMENT_ORDER`; штатный успех показывается в строке состояния без модального окна.
