## Workflow Rules

- For a new feature or non-trivial change, discuss and write the plan first, then implement.
- Before implementation, create a dedicated branch from the current `main`; do not implement the feature directly on `main`.
- If the branch passes checks without errors, commit there first and merge back into `main`.
- The next change starts from a new branch again; do not keep piling unrelated work into the previous feature branch.
